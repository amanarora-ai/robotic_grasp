# VegaPickYCB — Robotic Grasping Submission

Reinforcement learning for picking YCB objects with the Dexmate Vega dexterous hand in MuJoCo MJX. Single-policy grasping across four YCB objects.

See [REPORT.md](REPORT.md) for the full technical write-up.

## Setup

### 1. Create a Python environment

Using `conda` / `micromamba`:
```bash
micromamba create -n vega_grasp python=3.12 -y
micromamba activate vega_grasp
```

Or `venv`:
```bash
python3.12 -m venv vega_grasp
source vega_grasp/bin/activate
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

This installs JAX with CUDA 12 wheels (`jax[cuda12]==0.10.0`). If you see CUDA errors at runtime, verify your CUDA driver with `nvidia-smi` and re-install JAX following https://jax.readthedocs.io/en/latest/installation.html.

### 3. Install `mujoco_playground`

`mujoco_playground` is a companion library from Google DeepMind; we use it for the MJX env wrapper. Clone and pip-install as editable:

```bash
# from the parent directory of this submission/
git clone https://github.com/google-deepmind/mujoco_playground.git
pip install -e mujoco_playground
```

The training scripts add `mujoco_playground/` to `sys.path` via `sys.path.insert(0, os.path.join(BASE_DIR, "mujoco_playground"))`, so place the clone next to this `submission/` folder or in the parent directory so the path resolves. If you install it system-wide (pip install), that also works.

### 4. (Already done, do not re-run unless modifying URDF) URDF → MJCF conversion

The file `xmls/vega_right_arm.xml` is **already generated** from the Vega URDF and included in this folder. You only need to re-run the conversion if you modify the URDF or the actuator/site list:

```bash
python scripts/convert_urdf.py
```

### 5. Verify the scene loads

```bash
python scripts/verify_scene.py
```

This loads `xmls/vega_right_arm_scene.xml`, prints actuator/site info, steps the physics 100 steps for stability, and writes two videos:
- `scene_verify.mp4` — full scene, 10 seconds
- `scene_verify_hand_closeup.mp4` — hand close-up showing fingertip sites

If this step passes without error, the install is correct.

## Training

### SAC — single-object (reference run, reproduces the included best policy)

```bash
CUDA_VISIBLE_DEVICES=0 python scripts/train_sac_vmap.py \
    --name smallbuf_lowreach_v1 \
    --target-entropy -6.0 \
    --reach-scale 0.05 \
    --obj-touch-scale 2.0
```

Runtime: ~6–10 hours on a single A100 for 1 000 iterations. Checkpoints are written to `checkpoints/VegaPickYCB_SAC_vmap-<timestamp>-<name>/`.

### PPO — softgrasp reward

```bash
CUDA_VISIBLE_DEVICES=0 python scripts/train_ppo_softgrasp.py \
    --name softgrasp_v1 \
    --fixed-arm-init
```

This uses the softgrasp reward spec (multiplicative lift: `grasp_ok × height × time`, binary success at 5 cm / 5 steps). Reward scales can be overridden per-component, e.g. `--reach 0.5 --lift 3.0 --success 10.0`.

### SAC — multi-object (all 4 YCB objects in one policy)

The multi-object trainer (`scripts/train_sac_final.py`) is in active development; see REPORT.md §12 for the full NaN-stability history. Not reproduced in this submission because the run does not yet yield a general policy.

## Inspecting / Rendering a Trained Policy

The reference SAC policy is included. Render 4 episodes to mp4:

```bash
python scripts/render_policy.py \
    --ckpt checkpoints/smallbuf_lowreach_v1 \
    --tag best \
    --n 4 \
    --out demo.mp4
```

Arguments:
- `--ckpt` — run directory
- `--tag` — `best` (peak-success params) or `latest`
- `--n` — number of episodes
- `--out` — output mp4 path (default: auto-named under `videos/`)

## Troubleshooting

**`ModuleNotFoundError: mujoco_playground`** — you skipped step 3 of setup. Clone from GitHub and pip-install editable, OR put the clone next to `submission/`.

**`ImportError: cannot import name ...` from brax** — brax versions before 0.14.2 had a different gradient-update API; `pip install brax==0.14.2` exactly.

**`RuntimeError: Failed to create EGL context`** — for video rendering, MuJoCo needs EGL or OSMesa. The scripts set `MUJOCO_GL=egl` by default. On a headless server without EGL, use `MUJOCO_GL=osmesa` and install `osmesa-libs` (Ubuntu: `apt install libosmesa6`).

**`jaxlib.xla_extension.XlaRuntimeError: RESOURCE_EXHAUSTED`** — out of GPU memory. Reduce `--num-envs` in the training script (default 1024 for PPO, 256 for SAC) or lower `replay_capacity` in SAC_CFG.
